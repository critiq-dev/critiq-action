# Security policy — critiq-action

## Supported versions

Security fixes are applied to the default branch and released tags (for example `@v1`). Pin workflows to a tag or commit SHA for predictable behavior.

## Reporting a vulnerability

**Do not** open a public GitHub issue for undisclosed security vulnerabilities.

Report security issues for this repository through [GitHub Security Advisories](https://github.com/critiq-dev/critiq-action/security/advisories/new) (preferred) or by opening a private contact path via the [`critiq-dev`](https://github.com/critiq-dev) organization if Advisories are unavailable.

Include:

- A description of the issue and impact.
- Steps to reproduce or a proof of concept.
- Affected versions or commits.

We will acknowledge reports as soon as practicable and coordinate disclosure after a fix or mitigation is available.

## Scope

This policy covers the `critiq-dev/critiq-action` composite action and its scripts. Vulnerabilities in **`@critiq/cli`** or **`@critiq/rules`** should be reported to the relevant package repository ([`critiq-core`](https://github.com/critiq-dev/critiq-core), [`critiq-rules`](https://github.com/critiq-dev/critiq-rules)) as appropriate.
